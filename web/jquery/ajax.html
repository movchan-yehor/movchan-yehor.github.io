Що таке AJAX

AJAX — це механізм відправки HTTP-запитів зі сторінки без її перезавантаження. Сторінка "живе", а дані підтягуються у фоні. jQuery значно спрощує цей процес порівняно з чистим JS.

// Vanilla JS — довго
fetch('/api/data')
  .then(r => r.json())
  .then(data => {
    console.log(data);
  })
  .catch(err => {
    console.error(err);
  });

// jQuery — коротко
$.get('/api/data', function(data) {
  console.log(data);
}).fail(function(err) {
  console.error(err);
});
jQuery автоматично:
✓ парсить JSON
✓ обробляє помилки
✓ крос-браузерна підтримка
✓ зручний ланцюжок .done/.fail

$.get() — отримати дані

Найпростіший спосіб зробити GET-запит. Перший аргумент — URL, другий — callback з даними. jQuery автоматично парсить JSON якщо сервер повертає правильний Content-Type.

$.get() — реальний запит до публічного API
$.get(
  'https://api.github.com/users/jquery',
  function(data) {
    console.log(data.name);
    console.log(data.public_repos);
  }
);

// Скорочений запис
$.get(url, callback);

// З параметрами
$.get(url, { page: 1 }, callback);

$.post() — відправити дані

POST-запит для відправки даних на сервер. У навчальному проєкті ми не відправляємо реальних даних — але важливо знати синтаксис. Демо використовує jsonplaceholder.typicode.com — публічний тестовий сервер.

$.post() — тестовий запит
$.post(
  'https://jsonplaceholder.typicode.com'
  + '/posts',
  {
    title:  'Моя цитата',
    body:   'Текст цитати...',
    userId: 1
  },
  function(data) {
    console.log('створено:', data.id);
  }
);

$.ajax() — повний контроль

Базовий метод, на якому побудовані $.get() і $.post(). Дає повний контроль над запитом — метод, заголовки, тип даних, таймаут і callbacks для кожного стану.

$.ajax() з усіма опціями
$.ajax({
  url:      '/api/quotes',
  method:   'GET',
  dataType: 'json',
  timeout:  5000,

  beforeSend: function() {
    // перед запитом — показати лоадер
    $('#loader').show();
  },

  success: function(data) {
    // дані отримано
    renderCards(data);
  },

  error: function(xhr, status) {
    // щось пішло не так
    showError(status);
  },

  complete: function() {
    // завжди — сховати лоадер
    $('#loader').hide();
  }
});

Індикатор завантаження

Поки дані завантажуються — користувач має бачити що щось відбувається. Показуємо лоадер у beforeSend, ховаємо в complete.

патерн лоадера
$.ajax({
  url: './data/quotes.json',

    $('#loader').fadeIn(200);
    $('#board').hide();
  },

  success: function(data) {
    renderBoard(data);
    $('#board').fadeIn(400);
  },

  complete: function() {
    $('#loader').fadeOut(200);

// 06

Обробка помилок

Запит може не вдатись — неправильний URL, сервер недоступний, таймаут. Завжди обробляй помилки і показуй користувачу зрозуміле повідомлення.

обробка різних типів помилок
$.ajax({
  url:     url,
  timeout: 5000,

  error: function(xhr, status, err) {

    if (status === 'timeout') {
      showError('Сервер не відповідає');

    } else if (xhr.status === 404) {
      showError('Файл не знайдено');

    } else if (xhr.status === 0) {
      showError('Немає з\'єднання');

    } else {
      showError('Помилка: ' + xhr.status);
    }
  }
});

function showError(msg) {
  $('#error-msg')
    .text(msg)
    .slideDown(300);
}
запит...
Ніколи не залишай запит без обробки помилки. Навіть якщо просто — $.get(url, fn).fail(function() { alert('помилка') })

Робота з JSON-відповіддю

jQuery автоматично парсить JSON якщо dataType: 'json' або сервер повертає правильний Content-Type. Ти отримуєш готовий об'єкт або масив — одразу можна ітерувати.

отримати масив та відрендерити картки
$.get(
  'https://jsonplaceholder.typicode.com'
  + '/users?_limit=3',

  function(users) {
    // data — вже масив об'єктів
    $.each(users, function(i, user) {
      const $card = $('<div>')
        .addClass('card')
        .html(`
          <b>${user.name}</b>
          <span>${user.email}</span>
        `);

      $card.hide();
      $('#board').append($card);
      $card.fadeIn(300);
    });
  }
);

JSON-файл на GitHub

Для навчального проєкту найпростіший варіант — покласти JSON на GitHub і робити запит до raw-посилання. Безкоштовно, без реєстрації додаткових сервісів.

структура файлу та підключення
// quotes.json (твій файл на GitHub)
[
  {
    "id":     1,
    "text":   "Текст цитати...",
    "author": "Ім'я автора",
    "tag":    "мотивація",
    "likes":  0
  }
]

// Посилання для запиту:
// raw.githubusercontent.com/
//   USERNAME/REPO/main/quotes.json

const URL = 'https://raw.githubusercontent'
          + '.com/username/repo'
          + '/main/quotes.json';

$.get(URL, function(data) {
  renderBoard(data);
}).fail(function() {
  showError('Не вдалось завантажити');
});
1. Створи репозиторій на GitHub
2. Додай файл quotes.json
3. Відкрий файл → кнопка Raw
4. Скопіюй URL → вставай в $.get()
5. Готово — дані підвантажуються
Важливо про зміни: дані які користувач додає або видаляє — існують тільки в пам'яті браузера. Після перезавантаження сторінки знову підвантажаться початкові дані з JSON. Це нормально для навчального проєкту.
