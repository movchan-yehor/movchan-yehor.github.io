КласFileManager

Реалізуйте клас, який обгортає стандартні діалогові вікна filedialog і вирішує типові проблеми при роботі з файлами в Python.

01 / 04Службовий код

Читання і запис — один і той самий блок скрізь

filedialog повертає лише рядок зі шляхом. Далі розробник щоразу сам пише with open(…), задає кодування, обробляє винятки — і той самий блок з'являється в десяти різних місцях проєкту.

Проблема 
path = filedialog.askopenfilename()
if path:
    with open(path, 'r',
               encoding='utf-8') as f:
        content = f.read()
# ↑ той самий блок у кожній функції
Рішення 
content = fm.open_and_read()
# Повертає вміст файлу або None
02 / 04Активний файл

Втрата посилання на активний файл

Після відкриття файлу шлях живе у локальній змінній. Функція «Зберегти» не знає про нього — і знову запитує користувача через діалог, хоча він вже вибирав файл хвилину тому.

Проблема 
def open_and_read():
    path = filedialog.askopenfilename()
    # path живе тільки тут

def save_file():
    # path недоступний — знову діалог!
    path = filedialog.asksaveasfilename()
Рішення 
fm.open_and_read()       # відкрили, запам'ятали
fm.save_to_current()     # зберегли без діалогу
! current_file оновлюється при кожному успішному open_and_read() і стає None при close().
03 / 04Пам'ять

«Амнезія» діалогових вікон

Стандартні функції filedialog не запам'ятовують, де користувач був минулого разу. Кожен виклик відкриває вікно у домашній директорії або там, звідки запущено скрипт.

Проблема 
# Кожен виклик — з нуля
path = filedialog.askopenfilename()
# initialdir не зберігається між викликами
Рішення 
# Менеджер підставить останню папку
path = fm.open_and_read()
# _last_dir оновлюється автоматично
ℹ Клас зберігає _last_dir у своєму стані. При кожному успішному виборі поле оновлюється автоматично. Наступний виклик передає його як initialdir у стандартний діалог.
04 / 04Історія

Відсутність списку нещодавніх файлів

Список «Нещодавні файли» — стандартне очікування користувача. Реалізація вручну потребує глобального списку, уникнення дублікатів, обмеження розміру та оновлення UI — все це розкидано по різних місцях коду.

Проблема 
recent_files = []

def open_and_read():
    path = filedialog.askopenfilename()
    if path and path not in recent_files:
        recent_files.insert(0, path)
        if len(recent_files) > 10:
            recent_files.pop()
Рішення 
fm.open_and_read()  # автоматично → history

recent = fm.get_history()
# → list[str], до 10 унікальних,
#   найновіший — першим
ℹ Дублікати автоматично переміщуються на початок, а не додаються повторно. Скасований вибір до списку не потрапляє.
list[str] — до 10 унікальних шляхів, найновіший першим
